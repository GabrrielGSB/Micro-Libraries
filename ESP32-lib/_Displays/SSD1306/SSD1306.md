# Classe `SSD1306`

Driver para displays OLED baseados no controlador **SSD1306**. 

---

## Dependências

| Arquivo     | Papel                                                   |
|-------------|---------------------------------------------------------|
| `I2C.hpp`   | Classe base — inicializa e gerencia o barramento I2C    |
| `font.h`    | Matriz de pixels dos 95 caracteres ASCII imprimíveis    |

---

## Conceito de Framebuffer

O SSD1306 organiza sua memória em **páginas**: cada página é uma faixa horizontal de 8 pixels de altura. Um display 128×64 possui 8 páginas; um 128×32 possui 4.

O framebuffer é uma cópia dessa memória mantida na RAM do ESP32 (`1024 bytes` para 128×64). Todas as operações de desenho modificam apenas esse buffer. A tela física só é atualizada quando `atualizar()` é chamado explicitamente, evitando atualizações parciais via I2C.

```
Ciclo de uso:
  limpar() → desenharPixel() / escreverTexto() → atualizar()
```

---

## Interface da classe

```cpp
class SSD1306 : public I2C {
public:
    SSD1306(i2c_port_t porta_i2c,
            gpio_num_t pino_sda,
            gpio_num_t pino_scl,
            uint16_t larg  = 128,
            uint16_t alt   = 64,
            uint8_t  ender = 0x3C,
            uint32_t freq  = NORMAL_MODE);
    ~SSD1306();

    // Inicialização e energia
    void inicializar();
    void ligar();
    void desligar();
    void contraste(uint8_t nivel);
    void inverter(bool inverter_cores);

    // Framebuffer
    void limpar();
    void desenharPixel(int x, int y, bool ligado);
    void desenharChar(char caractere, int x, int y);
    void escreverTexto(const char* texto, int x, int y);

    // Envia o framebuffer para o display
    void atualizar();
};
```

## Como usar

### 1. Incluir o header

```cpp
#include "SSD1306.hpp"
```

Não é necessário incluir `I2C.hpp` separadamente — ele já é herdado.

### 2. Criar o display

O construtor recebe os pinos SDA e SCL diretamente e inicializa o barramento I2C internamente.

```cpp
// 128×64, endereço 0x3C, 100 kHz (padrão)
SSD1306 display(I2C_NUM_0, GPIO21, GPIO22);
```

Para displays com configurações diferentes:

```cpp
// 128×32, endereço alternativo 0x3D, fast mode
SSD1306 display(I2C_NUM_0, GPIO21, GPIO22, 128, 32, 0x3D, FAST_MODE);
```

### 3. Inicializar

```cpp
display.inicializar(); // obrigatório antes de qualquer uso
```

`inicializar()` envia a sequência completa de configuração ao hardware, limpa o framebuffer e chama `atualizar()` internamente — o display já aparece em branco após esta chamada.

---

## Métodos

### `inicializar()` — Configura e liga o display

Envia todos os comandos de setup (clock, multiplex, charge pump, contraste, etc.) e exibe uma tela limpa. Deve ser chamado uma única vez após a criação do objeto.

```cpp
display.inicializar();
```

---

### `ligar()` / `desligar()` — Controle de energia

Liga e desliga o painel sem apagar o conteúdo do framebuffer.

```cpp
display.desligar(); // apaga o painel (economia de energia)
display.ligar();    // reacende exibindo o conteúdo anterior
```

---

### `contraste(nivel)` — Ajusta o brilho

Valor de `0` (mínimo) a `255` (máximo). O padrão definido na inicialização é `0xFF`.

```cpp
display.contraste(128); // brilho médio
display.contraste(0);   // quase apagado
display.contraste(255); // brilho máximo
```

---

### `inverter(bool)` — Inverte as cores

```cpp
display.inverter(true);  // pixels ligados ficam apagados e vice-versa
display.inverter(false); // modo normal
```

---

### `limpar()` — Zera o framebuffer

Apaga todo o conteúdo da memória. O display físico não muda até `atualizar()` ser chamado.

```cpp
display.limpar();
display.atualizar(); // aplica o apagamento na tela
```

---

### `desenharPixel(x, y, ligado)` — Acende ou apaga um pixel

Coordenada `(0, 0)` é o canto superior esquerdo. Pixels fora dos limites da tela são ignorados silenciosamente.

```cpp
display.desenharPixel(0, 0, true);   // acende o pixel no canto superior esquerdo
display.desenharPixel(63, 31, true); // centro do display 128×64
display.desenharPixel(0, 0, false);  // apaga o pixel
```

---

### `desenharChar(caractere, x, y)` — Desenha um caractere

Renderiza um caractere ASCII imprimível (espaço `' '` até `'~'`) em fonte de **8×13 pixels**. A posição `(x, y)` é o canto inferior esquerdo do caractere.

```cpp
display.desenharChar('A', 10, 20);
display.desenharChar('!', 60, 50);
```

> Caracteres fora do intervalo ASCII 32–126 são ignorados.

---

### `escreverTexto(texto, x, y)` — Escreve uma string

Percorre a string e desenha cada caractere com avanço automático de **8 pixels** para a direita. Não faz quebra de linha automática (*strings longas ultrapassam a borda da tela*).

```cpp
display.escreverTexto("Ola, Mundo!", 0, 28);
display.escreverTexto("Temp: 24C", 0, 50);
```

---

### `atualizar()` — Envia o framebuffer ao display

Transmite os `largura × paginas` bytes do buffer via I2C para a memória do controlador SSD1306. Deve ser chamado após qualquer conjunto de operações de desenho para que as mudanças apareçam na tela.

```cpp
display.atualizar();
```

> Em um display 128×64, cada chamada transfere **1024 bytes** via I2C. Chame apenas quando o frame estiver completo, não a cada pixel.

## Exemplos completos

#### "Olá, Mundo!"

```cpp
#include "SSD1306.hpp"

extern "C" void app_main() {
    SSD1306 display(I2C_NUM_0, GPIO21, GPIO22);
    display.inicializar();

    display.limpar();
    display.escreverTexto("Ola, Mundo!", 0, 28);
    display.atualizar();
}
```

---

#### Contador em tempo real

```cpp
#include "SSD1306.hpp"
#include "Tempo.hpp"
#include <stdio.h>

extern "C" void app_main() {
    SSD1306 display(I2C_NUM_0, GPIO21, GPIO22);
    display.inicializar();

    char buf[32];
    int contador = 0;

    while (true) {
        snprintf(buf, sizeof(buf), "Count: %d", contador++);

        display.limpar();
        display.escreverTexto(buf, 0, 28);
        display.atualizar();

        delay_ms(500);
    }
}
```

### Desenhando formas com pixels

```cpp
#include "SSD1306.hpp"

extern "C" void app_main() {
    SSD1306 display(I2C_NUM_0, GPIO21, GPIO22);
    display.inicializar();

    display.limpar();

    // Borda superior e inferior
    for (int x = 0; x < 128; x++) {
        display.desenharPixel(x, 0,  true);
        display.desenharPixel(x, 63, true);
    }

    // Borda esquerda e direita
    for (int y = 0; y < 64; y++) {
        display.desenharPixel(0,   y, true);
        display.desenharPixel(127, y, true);
    }

    display.escreverTexto("ESP32", 44, 28);
    display.atualizar();
}
```

### Modo economia de energia

```cpp
display.desligar();  // apaga o painel sem perder o conteúdo
delay_s(30);
display.ligar();     // reexibe o último frame
```

### Compartilhando o barramento I2C com outro sensor

Como `SSD1306` herda de `I2C`, o barramento pode ser compartilhado com outros dispositivos passando a mesma porta e pinos — o barramento já existente é reutilizado automaticamente.

```cpp
#include "SSD1306.hpp"
#include "I2C.hpp"

extern "C" void app_main() {
    SSD1306 display(I2C_NUM_0, GPIO21, GPIO22); // cria o barramento
    I2C     sensor (I2C_NUM_0, GPIO21, GPIO22); // compartilha o mesmo

    display.inicializar();

    uint8_t raw[6];
    sensor.escrever(0x68, 0x6B, 0x00);         // acorda MPU-6050

    while (true) {
        sensor.ler(0x68, 0x3B, raw, 6);

        display.limpar();
        display.escreverTexto("Lendo IMU...", 0, 28);
        display.atualizar();

        delay_ms(100);
    }
}
```

## Resumo dos métodos

| Método              | Afeta display físico | Descrição                                    |
|---------------------|----------------------|----------------------------------------------|
| `inicializar()`     | ✅ Sim               | Configura hardware e exibe tela limpa        |
| `ligar()`           | ✅ Sim               | Liga o painel                                |
| `desligar()`        | ✅ Sim               | Desliga o painel (preserva framebuffer)      |
| `contraste(n)`      | ✅ Sim               | Ajusta brilho (0–255)                        |
| `inverter(bool)`    | ✅ Sim               | Inverte as cores do painel                   |
| `limpar()`          | ❌ Só buffer         | Zera o framebuffer em RAM                    |
| `desenharPixel()`   | ❌ Só buffer         | Liga/desliga um pixel no framebuffer         |
| `desenharChar()`    | ❌ Só buffer         | Renderiza um caractere no framebuffer        |
| `escreverTexto()`   | ❌ Só buffer         | Renderiza uma string no framebuffer          |
| `atualizar()`       | ✅ Sim               | Envia o framebuffer inteiro ao display       |
